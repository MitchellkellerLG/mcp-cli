# Issue Tracker

## Location

Issues are tracked on GitHub:
https://github.com/MitchellkellerLG/mcp-cli/issues

## Remote

```
origin  https://github.com/MitchellkellerLG/mcp-cli (fetch)
origin  https://github.com/MitchellkellerLG/mcp-cli (push)
```

## Conventions

- **Bug reports**: Include the full error output (stderr), the command that triggered it, and the OS/Bun version.
- **Feature requests**: State the user problem first, not the solution.
- **Daemon issues**: Set `MCP_DEBUG=1` and include debug output. Note whether `MCP_NO_DAEMON=1` bypasses the issue.
- **Config issues**: Redact secrets but include the full `mcpServers` structure.

## Triage Owner

The maintainer triages new issues within 48 hours. Agents can attempt issues labeled `ready-for-agent` without further approval.
